# AI热点日报 - 产品迭代优化方案

## 一、当前产品不足分析

### 1. **内容展示单一，缺乏层次感**
- 所有热点都是统一尺寸卡片，无法区分重大突发新闻和普通热点
- 没有"头条"区域，今日最重要的热点无法突出展示
- 信息密度低，用户无法快速抓住今日核心看点

### 2. **内容组织方式简单**
- 只有平铺列表，没有按重要性/热度分组
- 缺乏"今日要闻"提炼，用户需要逐行浏览
- 历史数据没有很好的归档和导航，只能一页一页翻

### 3. **交互体验基础但不够丰富**
- 只有搜索和分类筛选，缺乏更多维度的筛选（热度、来源、时间等）
- 没有收藏/点赞功能，用户无法保存感兴趣的内容
- 没有分享功能，不利于传播

### 4. **视觉呈现单调**
- 纯文字卡片，没有图片展示
- 缺乏视觉变化，长时间浏览容易疲劳
- 没有今日概要/导读区域

### 5. **内容价值挖掘不够**
- 只展示原始信息，没有归纳总结
- 热点之间缺乏关联，看不出行业趋势
- 没有热门讨论汇总，用户不知道大家在关注什么

---

## 二、核心设计思路：从"单一列表" → "分层展示"

**新理念**：不同用户有不同需求 - 有的用户想快速扫一眼今日发生了什么，有的用户想深入看每个热点。应该让内容分层，满足不同需求。

新的整体结构：

```
┌─────────────────────────────────┐
│  Header + Search + 主题切换       │
├─────────────────────────────────┤
│ 【今日焦点 / 头版热点】            │  ← 高亮展示 Top 3 重大热点（大卡片+图片）
│  更大版面，更完整摘要              │
├─────────────────────────────────┤
│ 【今日速读 / 五分钟概览】          │  ← 浓缩所有热点，快速扫完
│  紧凑列表，一键展开详情            │
├─────────────────────────────────┤
│ 分类筛选栏 + 排序切换             │
├─────────────────────────────────┤
│ 【热点卡片流】（原有功能保留优化）  │  ← 普通热点用卡片展示
├─────────────────────────────────┤
│ 侧边栏：热门标签 + 历史日期导航    │
└─────────────────────────────────┘
```

## 六、技术实现要点

### 1. 图片处理策略
- **GitHub项目**：抓取仓库的Open Graph图片（GitHub自动生成）
- **论文**：按分类使用默认占位图，或抓取arXiv封面
- **新闻/博客**：提取文章的预览图
- **无图处理**：根据热度值生成渐变背景（热度越高颜色越鲜明）

### 2. 收藏功能
- 存在浏览器 `localStorage`，无需后端服务
- 数据格式：`{ collectedItems: ['id1', 'id2', ...] }`
- 用户可在收藏夹查看已收藏内容

### 3. 性能考虑
- 图片懒加载，进入视口再加载
- 今日速读默认折叠，展开再加载详情
- 使用现有TailwindCSS，不引入额外UI框架

## 八、新增 AI 学习板块（需求点 2）

### 8.1 产品定位

**AI学习板块**是一个全新的内容板块，定位：**每日精选最火的AI短视频教程，方便用户碎片化学习AI技能**。

- 核心价值：让用户在看新闻热点之余，能发现优质的AI学习视频
- 内容来源：B站（优先中文内容，符合需求）
- 呈现形式：视频卡片流，点击直接跳转到B站播放
- 更新频率：每日更新，和热点日报一起爬取

### 8.2 为什么选B站？

- ✅ **中文内容丰富**：国内AI教程最多的平台，符合"优先中文"需求
- ✅ **短视频流行**：大多数教程是5-15分钟短视频，适合碎片化学习
- ✅ **热度排序透明**：播放量、点赞数、投币数都可以作为热度指标
- ✅ **无需自己托管视频**：只爬取元数据，跳转B站播放，节省流量和存储空间
- ✅ **社区活跃度高**：UP主更新频繁，每天都有新内容

### 8.6 前端路由实现

因为是静态单页网站，使用 Hash 路由实现页面切换：

- `/#/` → 热点日报首页（默认）
- `/#/learning` → AI学习板块

实现简单，无需后端支持：
```javascript
// 监听hash变化
window.addEventListener('hashchange', routeChanged);

// 根据hash显示对应板块
function routeChanged() {
  const route = window.location.hash || '#/';
  if (route === '#/learning') {
    showLearningPage();
  } else {
    showHotPage();
  }
}
```
